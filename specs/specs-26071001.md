# Specification

## About

- This app is a markdown document editor.  
- Its function and design is simple, neutral and minimalistic.

## Structure

- The markdown document is rendered in a central column, 1000px wide.  
- On either side of the central column is a margin, 25px wide each (making the total document width 1050px).  
- The margins are coloured.
- The page has a fixed header and footer.
- As the document scrolls it scrolls underneath the header and footer.
- There is a margin above the document that is twice the height of the header. So when the document is scrolled to the top, there is a gap between the top and the header, of the height of the header.
- There is a margin below the document that is half the height of the page height. So when the doucment is scrolled to the bottom, the bottom of the document is in the middle of the page. This margin height should be scaled as the page size changes.
- In the header, on the left is the name of the file. It is aligned to the left side of the document.
- In the header, on the right is where the buttons go. They are aligned to the right side of the document.

## Sections

- A section is a part of a markdown document.  
- It starts with a heading (any level, `#` to `######`).  
- It ends just before the next heading.  
- The very first section does not need to start with a heading.
- There is always a token section at the end of the document.  
- If the markdown document is empty, only the token section will be displayed.  
- The token section can be highlighted and edited like a normal section. When it is saved with text, it becomes a regular section, and a new empty token section is automatically appended at the bottom.
- The states of a section are:
  - "default" - When a section is simply rendered.  
  - "highlighted" - When a section has focus on it (via click or keyboard navigation).  
  - "editable" - When a section has entered into edit mode.

## Fonts

- There is a directory of fonts.
- Formatted markdown should use the Noto Sans font.
- Markdown for editing should use the Noto Sans Mono font.
- Button icons should use the Material Symbols Rounded font.
- The button icons that will be used are: menu, edit, check and close.

## Behaviour

- Clicking a section will put it in the highlighted state, only if no other section is in an editable state. The previous section that was in the highlighted state will no longer be in the highlighted state and will revert to the default state.
- When a section is in the highlighted state, an edit button appears in the header.
  - The edit button is a square, grey in colour, containing the icon "edit".
  - Clicking this edit button puts the section in the editable state.
- When a section enters the editable state, two buttons replace the edit button in the header.
  - The first is the save button, with the "check" icon, it should be green.
  - The second is the cancel button, with the "close" icon, it should be pink.
- If additional headings are added to the section, when it gets saved it splits into multiple sections (one for each heading).  
- If the heading is removed from a section, when it gets saved it gets merged with the previous section. If it is the first section and its heading is removed, it remains as a heading-less first section.
- If the very first second becomes empty, it merges with the one below.

## Style

- Colours should generally be pastel colours.  
- The webpage background should be a very light grey.  
- The document background should be white.  
- The default left margin background should be light grey.  
- Corners should be square for layout blocks (document, section containers, editor fields). Do not round-off corners. (Action buttons in the margin are an exception and should be round).
- There should be no visible borders (e.g., no dark border) around the editor input area.

#### When a section is in the highlighted state

- The margin background should be light blue.

#### When a section is in the editable state

- The margin background should be light blue (the exact same light blue as the highlighted state).  
- The section background, containing the markdown, should be a very very light grey.  
- The font should be monospaced.  
- The editor input area has no scroll bar.  
- It vertically expands to fit the content.  

## Keyboard shortcuts

- There is a modifier key.
- The modifier on Windows/Linux: CTRL  
- The modifier on macOS: CMD  
- In the below, the modifier is represented by: MOD

#### State: No section is in the highlighted state

- UP/DOWN : The centre-most section (closest to the center of the viewport) is highlighted.

#### State: A section is in the highlighted state

- UP/DOWN : Moved the highlighted state to the section up/down.  
- ENTER : Changes the section to the editable state.
- ESC : Unhighlights the section and reverts it to the default state.

#### State: A section is in the editable state

- ESC : Ignore changes and go back to a highlighted state.  
- MOD + ENTER : Save changes and go back to a highlighted state.

## Code

- The code should use the "markdown-it" library from the local project dependencies.
