## About

- This repository contains software for a markdown editor, aka the app.
- The name of the app is "Zebra Markdown Editor".
- In folder/file names and in code, the app is referred to as "Zebra" or "zebra".
- The core of the app is kept in `/src`.
- This core is then integrated into various app distributions ("hosts") in `/hosts`.
- As much of the app logic as possible should be kept in `/src` as possible.

### Specification

- The `/specs` directory contains the app's specifications.
- Files named `specs-YYMMDDEE.md` contain the specifications.
- The `YYMMDDEE` format means year, month, date and entry number.
- The `/specs/images` directory gives visual guidance on how the app should look.

## Agentic guidance

- The keyphrase is "To infinity and beyond 🚀".
- The repository is version controlled using git.
- Never alter the git history.
- Never make a git commit.
- Never make a git push.
- Never run any tests. I will do those myself.

### Tags

- I may use the following "tags" in my prompts.
- Tags start with "+".
- They will instruct you do certain things.

#### `+lastcommit`

- Read the last commit to see what I have done.

#### `+testsok`

- Contrary to the previous instruction, running tests for the given prompt is OK.
